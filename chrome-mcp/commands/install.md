너는 지금 **코딩을 전혀 모르는 수강생**의 PC에서 크롬 MCP를 설치해 주는 중이다.
설치가 끝나면 이 사람의 클로드코드가 크롬 브라우저를 직접 보고 조작할 수 있게 된다.
크롬 확장 파일은 이 plugin 안에 이미 들어 있다: **`${CLAUDE_PLUGIN_ROOT}/extension`**

## 지켜야 할 태도
- **전문용어를 쓰지 말 것.** "환경변수", "PATH", "패키지", "레지스트리" 같은 말 금지. 꼭 필요하면 한 줄로 풀어서 설명한다.
- **한 번에 한 단계만.** 여러 단계를 한꺼번에 쏟아붓지 않는다.
- 매 단계마다 **지금 무엇을 왜 하는지 한 줄**로 먼저 말한다.
- 명령을 실행하기 전에 "이제 ○○를 합니다"라고 알리고, 끝나면 "됐습니다"라고 확인해 준다.
- 이미 설치·등록돼 있다는 메시지가 나오면 **그 단계는 건너뛰고** 다음으로 간다. 실패가 아니라고 안심시켜 준다.
- 에러가 나면 **당황시키지 말고**, 무슨 뜻인지 한 줄로 풀어 주고 다음 대처를 제시한다.

---

## 1단계 — Node.js가 있는지 확인
`node -v` 를 실행한다.
- **숫자가 나오면**(예: `v22.14.0`) → Node.js가 이미 있다. 2단계로.
- **"명령을 찾을 수 없다" 류 에러**면 → Node.js가 없다. 수강생에게 안내한다:
  > "크롬 MCP는 Node.js라는 프로그램 위에서 돌아갑니다. 자동차의 엔진 같은 거예요. 지금 없으니 먼저 깔아야 합니다. 어렵지 않아요."
  1. https://nodejs.org 접속
  2. 큰 초록색 **LTS** 버튼 클릭 (옆의 Current는 누르지 말 것)
  3. 받아진 설치 파일 실행 → 창이 뜨면 **[Next]/[다음]만** 누른다(체크박스 건드리지 말 것)
  4. "기기를 변경하도록 허용?" 창엔 **[예]**
  5. 끝나면 [Finish]

  설치가 끝나면 **반드시** 이렇게 안내하고 여기서 멈춘다:
  > "Node.js 설치가 끝났습니다. 이제 **클로드코드를 완전히 껐다가 다시 켜 주세요.** 새로 켠 다음 `/chrome-mcp:install` 를 다시 입력하면 됩니다. 방금 깐 새 프로그램을 컴퓨터가 알아보려면 한 번 새로 시작해야 하거든요."

  **중요: 여기서 대화를 끝낸다.** 껐다 켜지 않으면 다음 단계가 반드시 실패한다.

---

## ⚠️ 윈도우에서 명령을 실행하는 규칙 (2~4단계에 계속 적용)
윈도우 수강생 PC에서는 **반드시 `.cmd` 를 붙인 형태**로 실행한다. 그냥 쓰면 실패한다.

| 쓰지 말 것 (윈도우) | 대신 이것 |
|---|---|
| `npm ...` | `npm.cmd ...` |
| `mcp-chrome-bridge ...` | `mcp-chrome-bridge.cmd ...` |

이유(수강생에게 설명하지 말 것): 윈도우 PowerShell은 `npm`을 `npm.ps1`로 먼저 해석하는데, 기본 보안설정이 `.ps1` 실행을 막는다(`running scripts is disabled`). `.cmd`를 붙이면 이 검사를 안 거친다.
**절대 하지 말 것:** 수강생에게 `Set-ExecutionPolicy`로 보안설정을 바꾸게 하는 것. `.cmd`로 해결되므로 PC 보안을 건드릴 이유가 없다. **맥은 `.cmd` 안 붙임.**

## 2단계 — 크롬 MCP 프로그램 설치
> "이제 크롬과 대화할 프로그램을 설치합니다. 1~2분쯤 걸려요."

- 윈도우: `npm.cmd install -g mcp-chrome-bridge`
- 맥: `npm install -g mcp-chrome-bridge`

글자가 우르르 올라가고 warning이 몇 줄 나와도 **에러가 아니면 정상**. 기다린다고 미리 알려 준다.

- 맥: `mcp-chrome-bridge register`

## 4단계 — 클로드코드에 연결
> "이제 클로드코드가 이 프로그램을 쓸 수 있게 연결합니다."

먼저 설치 위치를 찾는다 — 윈도우: `npm.cmd root -g` / 맥: `npm root -g`.
나온 경로 뒤에 `mcp-chrome-bridge/dist/mcp/mcp-server-stdio.js`를 붙인 **전체 경로**를 만들고, 큰따옴표로 감싸 실행한다:

```
claude mcp add chrome-mcp-stdio -s user -- node "여기에_전체_경로"
```
예시(윈도우, 경로는 사람마다 다르니 **위에서 직접 찾은 값**을 쓸 것):
`claude mcp add chrome-mcp-stdio -s user -- node "C:\Users\사용자이름\AppData\Roaming\npm\node_modules\mcp-chrome-bridge\dist\mcp\mcp-server-stdio.js"`

이미 등록돼 있다는 메시지가 나오면 그대로 두고 다음으로.

## 5단계 — 크롬에 확장 프로그램 넣기 (수강생이 직접 하는 부분)
여기서부터는 **네가 대신 해 줄 수 없다.** 크롬 화면에서 수강생이 직접 클릭해야 한다.

먼저 **확장 폴더의 실제 절대 경로**를 확인해서 수강생에게 알려 준다. 이 plugin 안의 다음 폴더다:
`${CLAUDE_PLUGIN_ROOT}/extension`
(이 변수를 실제 절대경로로 풀어서, 예: `C:\Users\...\extension` 형태로 알려 줄 것. 5번에서 이 경로가 필요하다.)

그다음 **반드시 한 번에 한 단계씩만** 말하고, 수강생이 "됐어"라고 하면 다음으로. 다섯 개를 한꺼번에 나열하지 말 것.

1. 크롬 주소창에 `chrome://extensions` 입력하고 Enter
2. 화면 **오른쪽 위**의 **"개발자 모드"** 스위치를 켠다
3. 왼쪽 위에 새로 생긴 **"압축해제된 확장 프로그램을 로드"** 버튼 클릭
4. 폴더 선택 창에서 위에서 알려 준 **`extension` 폴더**를 선택하고 [폴더 선택]
   - ⚠️ `extension` 폴더 **안으로 들어가지 말고**, `extension` 폴더를 **선택한 상태**로 확인
5. 크롬 오른쪽 위 **퍼즐 아이콘**(🧩) 클릭 → 방금 설치된 확장 클릭 → **[Connect]** 버튼 클릭

못 찾겠다고 하면 화면 어디를 보라고 위치를 더 자세히 설명해 준다.

## 6단계 — 마무리
전부 끝나면 이렇게 안내하고 끝낸다:
> "설치가 끝났습니다! 마지막 두 가지만요.
> 1. **클로드코드를 완전히 껐다가 다시 켜 주세요.** (안 하면 연결이 안 됩니다.)
> 2. 새로 켠 다음 **`내 크롬 탭 목록 보여줘`** 라고 입력해 보세요.
> 크롬 탭 목록이 주르륵 나오면 **설치 성공**입니다. 🎉"

## 자주 나오는 문제
| 증상 | 원인 | 대처 |
|---|---|---|
| `npm.ps1 cannot be loaded because running scripts is disabled` | 윈도우 기본 보안설정이 `.ps1` 막음 | **`.cmd` 붙여 재실행**. 보안설정은 **절대 바꾸지 마라** |
| `npm`을 못 찾음 | Node 설치 후 껐다 켜지 않음 | 클로드코드 완전히 껐다 켜기 |
| 탭 목록을 못 함 | 4단계 후 재시작 안 함 | 껐다 켜기 |
| 확장이 "연결 안 됨" | Connect 안 누름 | 5단계-5 다시 |
| 확장 폴더 골랐는데 오류 | `extension` 폴더 안으로 들어가 선택 | `extension` 폴더 자체를 선택 |

## 맥(Mac)
순서는 윈도우와 같다. 다만 이 안내는 **맥 실물에서는 검증되지 않았다.** 막히면 솔직히 "이 부분은 검증 안 됐습니다"라고 말하고, 화면을 캡처해 오픈채팅에 올리도록 안내한다.
